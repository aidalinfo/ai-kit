# AI Kit Memory Bank

Ce dossier regroupe les notes de contexte à source de vérité pour les assistants et contributeurs.

- `projectbrief.md` : objectifs, périmètre, contraintes et critères de succès.
- `techContext.md` : dépendances, contrats techniques, opérations de base.
- `systemPatterns.md` : architecture, frontières de paquets, règles d’évolution.
- `activeContext.md` : priorité du moment, décisions récentes, actions en cours.
- `productContext.md` : problème produit, segments utilisateurs, DX/UX attendue.
- `progress.md` : livrables, fronts actifs, risques et protocole de mise à jour.
- `context.md` : plan de support du `ctx` dans les workflows.
- `branchParallel.md` : plan d’exécution parallèle de branches.
- `mem.md` : plan d’intégration de mémoire longue durée.
- `toon.md` : plan d’intégration TOON pour payloads structurés.
- `refactor.md` : plan de refactor ciblé de `ServerKit`.

Règle d’usage : commencer par `projectbrief.md`, `techContext.md` et `systemPatterns.md` puis lire le plus petit nombre de fichiers supplémentaire nécessaire.
